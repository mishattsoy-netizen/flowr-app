### 1. User Request
User request: "add max widht like in chat"

### 2. Objective Reconstruction
Enforce max-width and label text truncation on inline pill buttons (`.inline-link-btn`) inside copied/pasted notes blocks, matching the chat UI pill constraint.

### 3. Strategic Reasoning
- **Why it occurred**: The chat UI uses a strict `max-w-[120px] truncate` utility class on the label span inside inline links. When copied to notes, the inline pills didn't have these styles or CSS constraints, causing long titles (such as "The Best Online Flower Delivery Service - The New York Times") to render in full, making the pills extremely wide and causing layout overflow or line wrapping.
- **Approach**: 
  - Define custom CSS rules inside `globals.css` targeting `.inline-link-btn > span:last-child` (the text label span of the pill) to set `max-width: 120px`, `overflow: hidden`, `text-overflow: ellipsis`, and `white-space: nowrap`. This ensures browser-level truncation irrespective of tailwind compilations.
  - Implement defense-in-depth classes `max-w-[120px] truncate` directly on the inner span markup of the anchor tags generated by the parser in `markdownBlocks.ts` and the citation builder in `ChatMessage.tsx`.

### 4. Detailed Blueprint
- **Files Involved**:
  - `src/app/globals.css` (defining style rules for pill text spans)
  - `src/lib/editor/markdownBlocks.ts` (adding styling classes to `inlineToHtml` link parser template)
  - `src/components/assistant/components/ChatMessage.tsx` (adding styling classes to remaining citation template)
