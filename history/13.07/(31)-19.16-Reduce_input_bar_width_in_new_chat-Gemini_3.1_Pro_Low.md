User request: "can you make message bar a bit less wide in new pages"

### 2. Objective Reconstruction
Reduce the maximum width of the AI Assistant input bar specifically when it is in the centered "new chat" state, making it slightly more compact, while keeping it wider when active.

### 3. Strategic Reasoning
The `AIAssistant` was globally wrapped in a `max-w-4xl` container, making the input bar quite wide even when centered in an empty screen. To fulfill the user's request, this width can be made conditional: `max-w-3xl` when centered (empty chat), and expanding to `max-w-4xl` when docked at the bottom (active chat). A CSS transition was also added to ensure it smoothly expands if the user types and sends a message.

### 4. Detailed Blueprint
- `src/components/chat/ChatPage.tsx`: Modify the container wrapping the `AIAssistant` to use conditional width classes based on `showBottomBar`.

### 5. Operational Trace
- Added the `cn()` utility to the wrapping div.
- Applied `transition-all duration-300` for a smooth expansion effect.
- Applied the conditional logic `showBottomBar ? "max-w-4xl" : "max-w-3xl"`.

### 6. Status Assessment
The input bar in the empty state is now narrower (`max-w-3xl`) and flawlessly expands when entering the active state. Issue resolved.
