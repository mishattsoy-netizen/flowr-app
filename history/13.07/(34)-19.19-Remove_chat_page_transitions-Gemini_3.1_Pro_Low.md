### 1. User request
User request: "when i switch betwwen old chats and new chats, i see scale and width transitions"

### 2. Objective Reconstruction
Remove the CSS transitions on the `AIAssistant` wrapper that were causing visual glitches when the user navigated quickly between old and new chats in the sidebar.

### 3. Strategic Reasoning
When applying the custom scaling and width constraints for the new chat hero section, `transition-all` and `transition-transform` were added to smoothly interpolate between the states. However, since the vertical positioning (`top` to `bottom`) jumps instantly, having the width and scale lag behind for 300ms looked broken. Furthermore, when the user switches entirely different chats via the sidebar, the component preserves state and runs the CSS transition, causing an undesirable morphing effect. Stripping these CSS transitions forces the styles to apply instantly, mirroring the layout teleportation and resolving the glitch.

### 4. Detailed Blueprint
- `src/components/chat/ChatPage.tsx`: Strip `transition-all`, `transition-transform`, and `duration-300` from the `AIAssistant` wrappers.

### 5. Operational Trace
- Edited `ChatPage.tsx` and removed the transition classes from both `cn()` wrapper arrays.

### 6. Status Assessment
The width and scale constraints now snap instantly just like the position layout, completely eliminating the visual lagging when switching chats. Issue resolved.
