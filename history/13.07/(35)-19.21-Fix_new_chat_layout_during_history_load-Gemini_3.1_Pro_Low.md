### 1. User request
User request: "when i swithc from new chats pages to sessions in history, while message history is loading, i still see messagebar with tites and buttons"

### 2. Objective Reconstruction
Fix an issue where the centered "New Chat" greeting and input bar momentarily flashed on the screen when loading an old chat from the history sidebar, appearing overlapping with the loading skeleton.

### 3. Strategic Reasoning
The `showBottomBar` state determines whether the input bar snaps to the bottom (for active chats) or stays in the center (for new chats). Previously, this was calculated solely based on `displayMessages.length > 0`. Because fetching an old chat takes a fraction of a second, `displayMessages` is momentarily empty during the loading phase. This caused the app to mistakenly render the empty "New Chat" centered view over the skeleton.
By adding `!isTempChat` to the condition, the app now understands that any non-temporary chat (saved history chats) should automatically snap the bar to the bottom, regardless of whether the messages have finished downloading yet.

### 4. Detailed Blueprint
- `src/components/chat/ChatPage.tsx`: Modify the `showBottomBar` boolean logic to include `!isTempChat`.

### 5. Operational Trace
- Altered the boolean declaration from `displayMessages.length > 0 || isAILoading` to `displayMessages.length > 0 || isAILoading || !isTempChat`.

### 6. Status Assessment
The input bar now instantly correctly docks to the bottom when switching to a historical chat, preventing the centered "New Chat" view from flashing during the loading skeleton phase. Issue resolved.
