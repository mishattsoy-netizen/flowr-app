User request: "buttons under message bar in new chats are hidden under it, not below it"

### 0. Date and time of the request
13.07 19:11

### 1. User request
User request: "buttons under message bar in new chats are hidden under it, not below it"

### 2. Objective Reconstruction
Fix the layout issue where the Quick Access Pills (buttons for /image, /search, etc.) were rendered underneath the absolutely positioned `AIAssistant` in an empty chat, obscuring them.

### 3. Strategic Reasoning
Because the `AIAssistant` was moved to an absolutely positioned container in `ChatPage.tsx` to allow instant transition, it no longer flowed properly with the centering logic inside `ChatConversation.tsx`. This caused the pills rendered in `ChatConversation.tsx` to fall out of alignment.
To fix this flawlessly across all screen sizes, the pills logic and rendering were migrated from `ChatConversation.tsx` directly into the `ChatPage.tsx` absolute wrapper. This ensures the pills are always perfectly bound to the bottom of the `AIAssistant` when centered, completely avoiding any positioning mismatch.

### 4. Detailed Blueprint
- `src/components/chat/ChatConversation.tsx`: Remove `QUICK_ACCESS_PILLS` data structure, click handler, and DOM elements.
- `src/components/chat/ChatPage.tsx`: Import the required icons, inject the `QUICK_ACCESS_PILLS` array, and render the buttons conditionally directly below the `AIAssistant` when `!showBottomBar`.

### 5. Operational Trace
- Deleted the pill mapping and the click handler from `ChatConversation.tsx`.
- Imported `ImageIcon`, `Globe`, `Telescope`, and `CheckSquare` into `ChatPage.tsx`.
- Added the pill array and mapped over it inside the `flex flex-col items-center` container holding the `AIAssistant`, using the global state directly to `setAssistantInput`.

### 6. Status Assessment
The quick access pills are now correctly rendering exactly beneath the `AIAssistant` bar in the center of the screen during a new chat. The issue is resolved.
