Three Approaches

How to add Excalidraw-style features to your existing canvas

A

⭐ Native SVG shape layer (recommended)

Add a new shape block type stored in the same Zustand store. Shapes are rendered as SVG elements in an overlay. New toolbar section for shape primitives. Style panel slides in on selection. Undo/redo via a history stack. Clean separation: existing blocks stay DOM-based, new shapes are SVG.

✓ No new dependencies ✓ Fits existing architecture ✓ Full control over look & feel ✗ More code to write
B

SVG + Rough.js (sketch style)

Same as A but uses roughjs to render shapes with the hand-drawn Excalidraw aesthetic. Since you said no hand-drawn style, this adds complexity without benefit.

✓ Exact Excalidraw look ✗ You didn't want hand-drawn style ✗ Extra dependency
C

Embed tldraw

Drop in the tldraw React component — a modern, clean alternative to Excalidraw with all the primitives, style panel, undo/redo, and export built in. Styled to match your app. Existing canvas becomes a separate mode.

✓ Instant full feature parity ✓ Actively maintained ✗ Large bundle (~1MB) ✗ Harder to integrate with your store

My recommendation is A — it fits your existing architecture, keeps the bundle lean, and gives you full control. Reply in the terminal.
